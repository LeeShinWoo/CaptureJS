운영체제
기능 : 시스템 자원을 효율적으로 사용하게 하는, 사용자와 컴퓨터 간의 인터페이스 역할

운영체제 구조
	1단계 : 프로세서 관리
	2단계 : 메모리 관리
	3단계 : 프로세스 관리
	4단계 : 주변장치 관리
	5단계 : 파일 관리

프로레서란? 컴퓨터에서 명령을 해독하고 실행하는 기능 단위라는 뜻으로, ‘중앙 처리 장치’를 이르는 말.

운영체제의 기술발전 및 흐름

	-1세대 : 일괄처리시스템
	-2세대 : 다중프로그램 시스템, 시분할 시스템, 다중처리 시스템, 실시간 시스템
	-3세대 : 다중모드처리
	-4세대 : 분산처리 시스템

프로세스 관리
PCB :
	운영체제가 프로세스에 대한 정보를 저장해 놓은 저장소
	다른 프로세스로 제어를 넘겨줄 떄, 현재 실행중인 프로세스의 정보를 PCB에 저장하고 제어를 넘김
	각 프로세스가 생성 될 때마다 PCB가 생성되고, 종료되면 제거됨
	부모 프로세스와 자식 프로세스는 PCB를 공유하지 않음

프로세스 구성 요소
	(Low Address)Code 영역/ Data영역/ Stack영역/ Heap영역(High Address)

프로세스 상태와 전이
	상태 : 생성/ 준비/ 실행/ 대기/ 종료
	상태의 전이 : Dispatch(준비->실행) / Timer Runout(실행->준비) / Block(실행->대기) / Wake Up(대기->준비)

프로세스 스케쥴링
	선점형 스케쥴링 정책 : 처리 도중 뺏기 가능
	-SJF(Shortest Job First , 최단작업 우선 처리)
	-SRT(Shortest Remaining Time First, 최단 잔여 시간)
	-RR(Round Robin, 순환할당 스케쥴링)
	-MLQ(Multi Level Queue, 다단계 큐)
	-MFQ(Multi level Feedvack Queue, 다단계 피드백 큐)

	비선점형 스케쥴링 정책 : 일산 사용권을 받으면 끝까지 쓰고 넘김
	-SJF(Shortest Job Firsr, 최단작업 우선 처리)
	-FIFO(First In First Out, 선입선출)
	-FCFS(First Come First Service, 선입선출)
	-HBN(Highest Response Ration Next) CPU할당을 받을 시간과 기다린 시간으로 계산되는 우선순위로 결정(Priority = runTime + waiting time / runTime) Priority 가 큰 수를 먼저 실행

프로세스 교착상태
	발생조건
	-상호 배제 : 한프로세스가 자원을 다른 프로세스가 사용 못하게 배제할때
	-점유와 대기 : 프로세스가 자신에게 할당된 자원을 해제 안하고 다른자월을 기다릴 때
	-비중단 조건 : 프로세스에 할당된 자월을 끝날 떄 까지 해제 할수 없을 때
	-환형대기 조건 : 상호 배제가 삼각 관계 이상일 때

	해결방안
	-예방 : 교착 상태의 필요조건을 부종함으로써, 교착 상태가 발생하지 않도록 하는 방법
	-회피 : 교착 상태 가능성을 인정하고, 적절하게 피하는 방법(은행원 알고리즘)
	-발견(탐지) : 교착 상태가 발생하면 교착 상태의 프로세스와 자원을 발견하는 방법
	-회복(복구) : 교착 상태를 일으킨 프로세스를 종료하고 할당된 자원을 회수하는 방법



기억장치 관리
	주기억 장치
	-반입기법(Fetch) : 언제 보조기억장치의 데이터를 주기억 장치로 옮기느냐?
		-요구 반입 정책
		-예상 반입 정책
	-배치기법(Placement) : 보조기억장치의 데이터를 주기억장치의 어디로 위치시키느냐?
		-최초 적합
		-최적 적합
		-최악 적합
	-교체기법(Replacement) : 공간확보를 위해 어떤 데이터를 제거 하고 새로운 데이터를 위치할 것이냐??
		-FIFO
		-LRU(Least Recently Used) : 가장 오랫동안 사용안한걸 바꿈
		-LFU(Least Frequently Used) : 가장 사용 횟수가 적은걸 바꿈
		-NUR(Not used Recently) : 최근에 전혀 사용되지 않은걸 바꿈
